Designing a CLI tool for managing my [[20250112080000]] Zettelkasten system. The tool needs to handle note creation, linking, and suggestion of related notes.

Commands to implement:
- `qn-new`: Create new note with timestamp ID
- `qn-link`: Bidirectional linking between notes
- `qn-suggest`: Find related notes based on [[20250110090000]] tags and content

Architecture decisions:
- Store notes in markdown format
- Use wiki-links `[[ID]]` for connections
- Build an index for fast lookups
- Implement TF-IDF for content similarity

This will integrate with my [[20250118103000]] neovim workflow eventually.
