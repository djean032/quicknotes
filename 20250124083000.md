# Database Options for Note Index

Created: 2025-01-24T08:30:00Z
Tags: database, design, json

---

Deciding how to store the index for my [[20250115090000]] QuickNotes system. Options:

**SQLite:**
- Fast queries
- ACID compliance
- Heavier dependency

**JSON file:**
- Portable, no dependencies
- Easy to debug
- Slower with large datasets

**LevelDB/Badger:**
- Key-value store
- Fast
- More complex API

For now, JSON seems fine. Can always migrate later if performance becomes an issue. The index is rebuilt periodically anyway.

Should document the [[20250130090000]] schema clearly.
